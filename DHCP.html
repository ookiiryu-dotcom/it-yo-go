DHCP Dynamic Host Configuration Protocol

ネットワークへ接続した端末に、IPアドレスや通信に必要な設定を
自動的に割り当てるためのプロトコル

DHCPとは？

DHCPとは、「Dynamic Host Configuration Protocol」の略で、 ネットワークへ接続したパソコンやスマートフォンなどに、 IPアドレスや通信に必要な設定を自動的に割り当てるプロトコルです。 プロトコルとは、機器同士が通信するための決まりを意味します。

家庭では主にルーターがDHCPサーバーとして働き、接続した端末へ設定を配ります。 企業では専用サーバーやネットワーク機器がこの役割を担う場合があります。 このページではDHCPv4を中心に説明し、IPv6との違いは後半で扱います。

DHCPの役割はインターネット回線を提供することではなく、そのネットワーク内で端末が通信を始められるように設定を自動化することです。

まずはイメージで理解する

DHCPは、ネットワークの「受付係」に例えられます。 新しく来た端末から利用申請を受け、空いているIPアドレスと必要な案内を一定期間貸し出します。

IPアドレスはネットワーク上の住所に当たります。 受付係が同じ住所を複数の端末へ誤って渡さないように管理することで、 利用者が端末ごとに設定しなくても通信を始められます。

例えばスマートフォンを自宅のWi-Fiへ初めて接続すると、通常はIPアドレスを手入力しなくても、ルーターから必要な設定を受け取れます。

なぜDHCPが必要なのか

DHCPがなければ、管理者や利用者が端末ごとにIPアドレス、 サブネットマスク、デフォルトゲートウェイ、DNSサーバーなどを設定する必要があります。 端末が増えるほど作業量が増え、入力ミスも起こりやすくなります。

また、同じIPアドレスを複数の端末へ設定すると、IPアドレスの重複によって通信できない場合があります。 DHCPは利用中のアドレスと貸し出し期限を管理し、利用可能なアドレスを再利用しやすくします。

DHCPv4の仕組み

DHCPv4の初回取得では、端末側のDHCPクライアントとDHCPサーバーが Discover、Offer、Request、ACKの順にメッセージを交換します。 頭文字を取って「DORA」と呼ばれる一般的な覚え方があります。

端末は最初、自分のIPアドレスもDHCPサーバーの場所も分かりません。 そのためDiscoverでは送信元を「0.0.0.0」、宛先を「255.255.255.255」とし、 同じネットワーク内へブロードキャストで問い合わせます。

1. Discover
端末が利用できるDHCPサーバーを
ブロードキャストで探す
↓
2. Offer
サーバーが利用可能なIPアドレスと
設定内容を端末へ提案する
↓
3. Request
端末が採用する提案とサーバーを
ブロードキャストで知らせる
↓
4. ACK
選ばれたサーバーが貸し出しを確定し、
通信に必要な設定を渡す

初回のRequestをブロードキャストすることで、複数のサーバーからOfferを受けた場合に、 選ばれなかったサーバーにも提案が採用されなかったことが伝わります。 一方、通常のリース更新では、端末は原則として貸し出し元のサーバーへ直接Requestを送ります。

ブロードキャストは通常、ルーターを越えません。 DHCPサーバーが別のネットワークにある場合は、DHCPリレーが端末とサーバーの間でメッセージを中継します。

DHCPサーバー

貸し出す範囲と設定を管理し、

異なるネットワークにいる端末と
サーバーの通信を中継します。

IPアドレスは永久に与えられるとは限らず、通常はリース期間を設けて貸し出されます。 端末は期限が切れる前に更新を試み、利用されなくなったアドレスは期限後に再び割り当てられます。 そのため、同じ端末でも再接続や設定変更によってIPアドレスが変わる場合があります。

DHCPサーバーは、端末から送られるクライアント識別子を使って端末を区別し、それがない場合はDHCPメッセージ内のハードウェアアドレスなどを使用します。

主な用途とメリット

DHCPは、家庭のWi-Fiから多数の端末を扱う企業ネットワークまで利用されます。 接続する端末が変わる環境では、必要な設定を自動化し、管理を一か所へ集められることが大きな利点です。

変更を反映しやすい

新しい内容を端末へ渡せます。

例えば会社のDNSサーバーを変更する場合、DHCPの配布設定を変更すれば、端末はリース更新や再接続の際に新しい情報を取得できます。

注意点と実務上の確認ポイント

DHCPは設定作業を減らしますが、サーバーや配布範囲の設計が不適切だと、 IPアドレスを取得できない、誤った通信先が設定される、 重要な機器のアドレスが変わるといった問題が起こります。

また、基本的なDHCPの交換だけで、接続した端末や応答したサーバーが信頼できることを保証するわけではありません。 接続制御、監視、ネットワーク分離などと組み合わせて運用します。

ルーターをアクセスポイントとして追加する場合、意図的な構成でなければ追加機器側のDHCPサーバー機能を停止し、既存のDHCPサーバーへ一本化します。

DHCPサーバーが停止しても、取得済みの端末はリース期限まで通信を続けられる場合がありますが、新規取得や更新は失敗する可能性があります。

固定IP・DHCP予約・IPv6との関係

常に同じIPアドレスが必要なプリンターやサーバーでは、 端末へ直接設定する固定IPアドレスか、DHCPサーバー側で同じアドレスを配るDHCP予約を使います。 どちらを選ぶかは、機器の用途や障害時の運用方法によって異なります。

まとめ

DHCPとは、ネットワークへ接続した端末に、IPアドレス、サブネットマスク、 デフォルトゲートウェイ、DNSサーバーなどの設定を自動的に割り当てるプロトコルです。

DHCPv4の初回取得では、Discover、Offer、Request、ACKの順に情報を交換します。 IPアドレスは通常、リース期間を設けて貸し出され、期限前に更新されます。

導入時には、配布範囲、固定設定との重複、サーバー停止時の影響、 DHCPリレー、貸し出し履歴、不正なDHCPサーバーへの対策を確認します。 同じIPアドレスが必要な機器では、固定IPアドレスとDHCP予約を使い分けます。
