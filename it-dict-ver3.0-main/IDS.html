IDS
（Intrusion Detection System）

不正アクセスや攻撃を検知する監視システム

IDSとは？

IDS（Intrusion Detection System） というのは、 ネットワークやシステム上の通信・動作を監視し、 不正アクセスや攻撃の兆候を検知して通知する仕組み です。

IDSは主に 「検知と可視化」 に重点を置いており、 ログ や アラート を通じて管理者へ状況を知らせます。

IDSは 通信そのものを自動で遮断しない 点が、 IPSとの大きな違いです。

IDSが行うこと

・不正な通信や攻撃の検知

・ログの収集・保存

・アラート通知 （メールや管理コンソールへの通知など）

・可視化・分析 （ダッシュボードやレポートで状況を把握）

IDSは、 「攻撃を防ぐ」よりも「異常に気づく」 ことを得意としています。

IDSの検知方法

IDSで使われる検知方法は、 IPSとよく似ています。

・シグネチャベース
既知の攻撃パターンと照合して検知する方式。

・アノマリーベース
通常とは異なる 異常なトラフィック や振る舞いを検知する方式。

・ポリシーベース
定義したルールに違反する通信を検知する方式。

IDSの種類

IDSは、配置場所によって役割が少し変わります。

・ネットワークIDS（NIDS）
スイッチやルーターの近くで ネットワークトラフィック を監視するタイプ。

・ホストIDS（HIDS）
サーバーや端末にエージェントを入れて、 OSやアプリケーションのログ・動作 を監視するタイプ。

IDSとIPSの違い

項目 IDS IPS
主な役割 検知・通知 検知・遮断
通信の遮断 行わない 自動で遮断する
目的 異常の発見 攻撃の防御

まとめ

IDSは、 ネットワークやシステムを監視し、 異常や攻撃を早期発見するための仕組み です。

SIEMやSOCなどと組み合わせて利用されることも多く、 セキュリティインシデントの早期発見に役立ちます。

IDSは、 「何が起きているかを把握するための目と耳」 のような存在です。
